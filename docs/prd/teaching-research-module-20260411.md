## 2. 背景与目标
当前系统已完成认证、用户、通讯录等基础能力，但教研教学模块尚未落地。根据早上确认的需求，第一批先聚焦于“专业、年级、课程、实施性教学计划”四类核心业务对象，建立稳定的数据结构和服务端接口。

本批不追求完整业务闭环，而是优先完成后端基础设施，确保后续前端开发、Excel 模板处理、导入导出、审批与统计能够基于统一的数据底座推进。

### 第一批目标
1. 建立专业、年级、课程、实施性教学计划的数据模型。
2. 落地对应后端 API 能力。
3. 固化核心业务规则，尤其是课程唯一性与删除限制。
4. 完成 OpenSpec 文档与数据库迁移，为下一批继续开发提供依据。

## 4. 核心业务对象

### 4.1 专业（Major）
用于表达学校内的专业主数据。

字段方向：
- 名称
- 学制（`THREE_YEAR` / `FIVE_YEAR`）
- 可选 ZIP 附件（后续批次可扩展）
- 状态

第一批要求：
- 支持基础 CRUD
- 支持状态管理
- 可作为年级与专业课的归属维度

### 4.2 年级（Grade）
用于表达某专业某届学生群体。

字段方向：
- 归属专业
- 学制
- 状态（`ACTIVE` / `GRADUATED`）
- `graduatedAt`（可空）

第一批要求：
- 必须绑定专业
- 可绑定实施性教学计划
- 已毕业 / 未毕业状态用于课程删除规则判断

### 4.3 课程（Course）
用于维护课程主数据。

课程分类口径：
- **公共课（PUBLIC）**：按课程名称唯一
- **专业课（MAJOR）**：按“专业 + 课程名称”唯一

附加维度：
- 来源：`MANUAL` / `PLAN_IMPORT`
- 状态：`ACTIVE` / `DISABLED`

第一批要求：
- 支持基础 CRUD
- 支持唯一性校验
- 支持删除限制与停用逻辑

### 4.4 实施性教学计划（TeachingPlan）
用于描述某年级某版本的教学计划。

第一批要求：
- 绑定年级
- 支持版本区分
- 支持基础 CRUD

### 4.5 实施性教学计划行（TeachingPlanRow）
用于表达计划中的课程明细。

字段方向：
- 所属教学计划
- 学期号 `termNo`
- 学期类型 `termType`（`SCHOOL` / `INTERNSHIP`）
- 课程名称 `courseName`
- 课程引用 `courseId`（可空）
- `weeklyHoursRaw`
- `weeklyHoursValue`（可空）
- `teacherName`
- `remark`
- `sortOrder`

## 5. 已锁定业务规则

### 5.1 学制规则
- **三年制**：6 学期（前 4 学期在校 + 后 2 学期实习）
- **五年制**：10 学期（前 8 学期在校 + 后 2 学期实习）
- 实习学期固定为企业实习，按 `30 节 × 18 周` 口径表达（导入导出与前端校验在后续批次处理）

### 5.2 课程唯一性规则
- **公共课**：按 `name` 唯一
- **专业课**：按 `(majorId, name)` 唯一
- 专业课必须绑定 `majorId`

### 5.3 课程删除规则
- 未被任何计划引用：允许删除
- 仅被已毕业年级引用：允许删除
- 被未毕业年级引用：禁止物理删除，并提示“只能停用”

### 5.4 第一批计划管理规则
- 网页编辑中的 30 节规则，第一批若涉及，只做提示思路，不在本批阻断
- 严格导入阻断、模板保真、合并单元格等 Excel 相关能力，留待后续批次
