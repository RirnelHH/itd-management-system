# 教研教学模块 PRD（第二批：前端基础管理页）

## 1. 文档信息
- **项目**：信息技术与设计系管理系统
- **模块**：教研教学模块
- **批次**：第二批
- **日期**：2026-04-11
- **目标**：将第一批已落地的后端能力接成可操作的前端基础管理页面。

---

## 2. 背景
第一批已完成教研教学模块的后端数据模型、NestJS API、Prisma migration 与关键业务测试，已经具备专业、年级、课程、实施性教学计划的服务端能力。

当前缺口不在后端，而在前端：系统还无法通过页面直接管理这些对象。因此第二批应聚焦“前端基础管理页”，先让专业、年级、课程、教学计划具备基础的增删改查能力。

---

## 3. 第二批目标
1. 建立教研教学模块的前端入口页面。
2. 打通第一批后端 API 与前端页面。
3. 提供基础资料管理体验，而不是继续扩展 Excel 或复杂计划编辑器。
4. 为第三批“教学计划网页编辑器”打基础。

---

## 4. 范围

### 4.1 包含范围
本批包含以下页面与能力：
- 专业管理页
- 年级管理页
- 课程管理页
- 教学计划列表页
- 基础表单弹窗 / 抽屉
- API 联调
- 前端构建验证

### 4.2 不包含范围
本批明确不包含：
- Excel 导入导出
- 教学计划明细行复杂编辑器
- 打印 / 报表 / 可视化
- 模板保真
- 审批流 / 通知流
- 复杂拖拽排序

---

## 5. 页面定义

### 5.1 专业管理页
#### 页面目标
让管理员可以维护专业基础资料。

#### 需支持
- 专业列表展示
- 新增专业
- 编辑专业
- 学制展示
- 删除操作（若存在关联数据则展示后端返回提示）

#### 页面字段
- 专业名称
- 学制（3年制 / 5年制）

---

### 5.2 年级管理页
#### 页面目标
让管理员可以维护年级，并与专业绑定。

#### 需支持
- 年级列表展示
- 按专业筛选
- 新增年级
- 编辑年级
- 毕业状态展示
- 删除操作

#### 页面字段
- 年级名称
- 归属专业
- 学制
- 状态
- 毕业时间

---

### 5.3 课程管理页
#### 页面目标
让管理员可以管理公共课与专业课。

#### 需支持
- 课程列表展示
- 按课程类型筛选
- 按专业筛选
- 新增课程
- 编辑课程
- 删除课程
- 状态展示

#### 页面字段
- 课程名称
- 课程类型（公共课 / 专业课）
- 归属专业（公共课为空）
- 来源类型
- 状态

#### 特殊交互规则
- 选择“公共课”时，隐藏专业选择
- 选择“专业课”时，必须选择专业
- 删除失败时展示后端业务提示

---

### 5.4 教学计划列表页
#### 页面目标
让管理员可以查看和创建教学计划，为第三批详情编辑页做入口准备。

#### 需支持
- 教学计划列表展示
- 按年级筛选
- 创建教学计划
- 查看详情入口（即使详情页暂未完全实现，也要保留入口占位）

#### 页面字段
- 教学计划名称
- 所属年级
- 所属专业
- 行数
- 创建时间

---

## 6. 交互原则
1. 保持当前系统风格一致。
2. 优先复用现有后台管理页面样式和组件。
3. 先保证“能管理”，再优化“好不好看”。
4. 错误提示以后端返回信息为准。

---

## 7. 验收标准
第二批完成时，应满足：
1. 页面入口可访问。
2. 四类页面可正常拉取数据。
3. 新增 / 编辑 / 删除基本流程可跑通。
4. 课程类型与专业联动逻辑正确。
5. 前端 build 通过。
6. 至少完成基础联调验证。

---
