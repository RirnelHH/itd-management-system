# 教研教学前端基础管理页规范

## ADDED Requirements

### Requirement: 专业管理页
系统 MUST 提供专业管理页，用于维护专业基础资料。

#### Scenario: 查看专业列表
- WHEN 用户进入专业管理页
- THEN 系统应展示专业列表
- AND 列表至少包含名称与学制

#### Scenario: 新增专业
- WHEN 用户提交新专业表单
- THEN 系统应调用后端创建接口
- AND 创建成功后刷新列表

### Requirement: 年级管理页
系统 MUST 提供年级管理页，用于维护年级资料并绑定专业。

#### Scenario: 按专业筛选年级
- WHEN 用户在年级页选择某个专业
- THEN 系统应仅展示对应专业的年级列表

#### Scenario: 展示毕业状态
- WHEN 用户查看年级列表
- THEN 系统应能看到年级当前状态与毕业时间

### Requirement: 课程管理页
系统 MUST 提供课程管理页，用于维护公共课与专业课。

#### Scenario: 公共课隐藏专业选择
- WHEN 用户在课程表单中选择公共课
- THEN 系统不应要求选择专业

#### Scenario: 专业课必须选择专业
- WHEN 用户在课程表单中选择专业课
- THEN 系统应要求选择专业

#### Scenario: 删除失败时展示后端业务提示
- WHEN 用户删除课程但后端返回业务错误
- THEN 前端应向用户展示后端返回的错误信息

### Requirement: 教学计划列表页
系统 MUST 提供教学计划列表页，作为后续详情编辑入口。

#### Scenario: 查看教学计划列表
- WHEN 用户进入教学计划页
- THEN 系统应展示教学计划列表
- AND 列表至少包含名称、所属年级、所属专业、行数

#### Scenario: 保留详情入口
- WHEN 用户查看教学计划列表
- THEN 系统应可见教学计划详情入口
- AND 即使详情页尚未完善，也应保留后续跳转位
