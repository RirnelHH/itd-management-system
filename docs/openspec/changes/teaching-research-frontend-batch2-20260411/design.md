# 教研教学模块第二批设计

## 设计目标
以最小可用原则，将第一批后端能力映射为后台前端页面，先实现“能管理”，再为第三批复杂编辑器铺路。

## 页面范围
- Majors 页面：专业资料管理
- Grades 页面：年级资料管理
- Courses 页面：课程资料管理
- Teaching Plans 页面：教学计划列表管理

## 设计原则
1. 复用现有前端管理台风格
2. 优先表格 + 弹窗表单模式
3. 保持 API 调用统一封装
4. 错误提示以服务端业务信息为准

## 数据交互
- 通过现有 `/api/v1` 前缀访问后端接口
- 页面初始化时加载列表数据
- 表单提交后刷新当前列表
- 删除失败时显示业务错误消息

## 扩展边界
本批不做教学计划详情编辑器，仅保留入口，为第三批继续实现。
