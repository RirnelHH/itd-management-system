# 修复登录态恢复、个人资料更新、通讯录访问异常

**变更日期**: 2026-04-10
**变更类型**: Bug 修复
**影响范围**: 前端登录态恢复、个人中心资料更新、通讯录 API 访问

---

## 问题描述

当前存在三个关联问题：

1. **刷新页面后退出登录**
   - 页面刷新后 token 虽可能仍在本地，但前端状态恢复不完整，表现为重新回到未登录状态或权限丢失。

2. **修改姓名后提示“用户不存在”**
   - 用户在个人中心修改姓名后，后续获取个人资料或访问需要鉴权的页面时出现“用户不存在”。
   - 需检查 profile/update/token payload/user lookup 整条链路是否存在 userId 丢失或取值错误。

3. **通讯录访问报 404**
   - 通讯录页面请求失败，需确认是前端请求路径错误、Nginx 代理问题、接口鉴权状态处理错误，还是前端对错误码的误判。

---

## 修复目标

- 刷新页面后，登录态可自动恢复，用户无需重新登录。
- 修改个人资料（至少姓名）后，用户仍能正常获取 profile，不出现“用户不存在”。
- 通讯录页面在登录状态下可以正常请求并展示公开用户数据。
- 前后端错误语义清晰：401/404/业务错误不混淆。

---

## 计划修改

### 前端
- 检查 `auth store` 的 token / userInfo 恢复逻辑
- 检查页面刷新时的 profile 拉取时机
- 检查个人中心修改资料后的本地状态同步逻辑
- 检查通讯录 API 请求路径、拦截器、异常处理

### 后端
- 检查 JWT 校验后的用户查询逻辑
- 检查 profile/update 接口是否使用了错误的 userId 字段
- 检查 users/public 接口路由、鉴权要求与返回语义
- 必要时补充更明确的日志和异常信息

### 部署/代理
- 检查 Nginx `/api/` 代理路径与 `VITE_API_BASE_URL` 是否一致
